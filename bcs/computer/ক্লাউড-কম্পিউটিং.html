সংজ্ঞা ও পরিচিতি

NIST (National Institute of Standards and Technology) এর মতে, ক্লাউড কম্পিউটিং হলো এমন একটি মডেল, যার মাধ্যমে ইন্টারনেটের সাহায্যে সার্ভার, স্টোরেজ, নেটওয়ার্ক, সফটওয়্যারসহ বিভিন্ন কম্পিউটিং রিসোর্স চাহিদামতো, দ্রুত এবং ন্যূনতম ব্যবস্থাপনা প্রচেষ্টায় ব্যবহার করা যায়।

NIST-এর ৫টি Essential Characteristics অবশ্যই মনে রাখুন

🚨
পরীক্ষায় সরাসরি আসে: "NIST অনুযায়ী cloud computing-এর essential characteristics কয়টি?" — উত্তর সবসময় ৫টি।
০১

On-Demand Self-Service

মানবীয় সহায়তা ছাড়াই ব্যবহারকারী নিজে computing resource নিতে পারে।

০২

Broad Network Access

মোবাইল, ল্যাপটপ, ট্যাবসহ বিভিন্ন ডিভাইস থেকে ইন্টারনেটের মাধ্যমে অ্যাক্সেস।

০৩

Resource Pooling

একই infrastructure একাধিক user একসাথে শেয়ার করে ব্যবহার করে।

০৪

Rapid Elasticity

চাহিদা বাড়লে দ্রুত resource বাড়ানো, কমলে কমানো যায়।

০৫

Measured Service

ব্যবহার অনুযায়ী বিল — এটাই Pay as you go মডেল।

⭐ মনে রাখার কৌশল

  • On-Demand = নিজে নিতে পারে | Broad Access = যেকোনো ডিভাইস
  • Resource Pooling = শেয়ারড ইনফ্রাস্ট্রাকচার | Elasticity = বাড়া-কমা
  • Measured Service = Pay as you go — বিদ্যুৎ বিলের মতো
🔧

তিনটি Service Model BCS-এ বারবার আসে

IaaS

Infrastructure as a Service

সার্ভার, স্টোরেজ, নেটওয়ার্ক ইত্যাদি অবকাঠামো ভাড়া নেওয়া হয়। ব্যবহারকারী নিজে OS, application ও data manage করে।
উদাহরণ: Amazon EC2, Azure Virtual Machines, Google Compute Engine

PaaS

Platform as a Service

Developer-দের জন্য ready-made platform — infrastructure provider manage করে। শুধু code লিখতে হয়।
উদাহরণ: Google App Engine, Microsoft Azure App Service, Heroku

SaaS

Software as a Service

সম্পূর্ণ software ইন্টারনেটের মাধ্যমে সরবরাহ করা হয়। ব্যবহারকারী শুধু ব্যবহার করে।
উদাহরণ: Gmail, Google Docs, Zoom, Microsoft 365, Salesforce

Modelপূর্ণ নামকে Manage করেউদাহরণপরীক্ষায়

চার ধরনের Deployment Model

🌍 Public Cloud

  • সাধারণ মানুষের জন্য উন্মুক্ত
  • অনেক user একই infrastructure share করে
  • সবচেয়ে সাশ্রয়ী
  • উদাহরণ: AWS, Azure, Google Cloud

🏢 Private Cloud

  • শুধু একটি প্রতিষ্ঠানের জন্য
  • সবচেয়ে নিরাপদ ও নিয়ন্ত্রিত
  • ব্যাংক ও সরকারি প্রতিষ্ঠানে ব্যবহৃত
  • খরচ বেশি

🔀 Hybrid Cloud

  • Public ও Private cloud-এর সমন্বয়
  • Sensitive data private-এ রাখা হয়
  • সাধারণ কাজ public-এ করা হয়
  • বর্তমানে সবচেয়ে practical model

🤝 Community Cloud

  • একই ধরনের চাহিদার একাধিক প্রতিষ্ঠান
  • যেমন: কয়েকটি বিশ্ববিদ্যালয়, হাসপাতাল
  • খরচ ও resource ভাগ করা যায়

পরীক্ষায় আসার মতো Key Terms

⚙️

Virtualization

একটি physical server-কে একাধিক virtual machine-এ ভাগ করার প্রযুক্তি। Cloud computing-এর মূল ভিত্তি।

📦

Containerization

Application-কে container-এ package করে যেকোনো environment-এ চালানো যায়। উদাহরণ: Docker, Kubernetes।

🔁

Scalability vs Elasticity

Scalability = resource বাড়ানো (manual);
Elasticity = demand অনুযায়ী স্বয়ংক্রিয়ভাবে বাড়া-কমা।

🌐

CDN

Content Delivery Network — বিভিন্ন জায়গায় server রেখে নিকটবর্তী location থেকে content দ্রুত সরবরাহ করে।

📜

SLA

Service Level Agreement — provider ও user-এর মধ্যে service quality, uptime ইত্যাদি নিয়ে চুক্তি।

🤖

Serverless Computing

Server manage না করেই code run করা যায়। Code চললে তবেই bill হয়। উদাহরণ: AWS Lambda।

💧

Utility Computing

পানি-বিদ্যুতের মতো computing service সরবরাহ। John McCarthy এই ধারণা দেন। Measured Service-এর ভিত্তি।

📡

Edge Computing

Data processing user-এর কাছাকাছি স্থানে করা হয়। IoT ডিভাইসে খুব গুরুত্বপূর্ণ। Low latency নিশ্চিত করে।

🔗

Vendor Lock-in

এক cloud provider থেকে অন্য provider-এ যাওয়া কঠিন হওয়া। Migration cost ও compatibility সমস্যা।

🏘️

Multi-tenancy

অনেক user একই মূল resource ও infrastructure share করে। Resource Pooling-এর মূল ধারণা।

⭐

বাংলা প্যারাগ্রাফ
ক্লাউড কম্পিউটিং হলো আধুনিক তথ্যপ্রযুক্তির একটি যুগান্তকারী ধারণা, যেখানে ইন্টারনেটের মাধ্যমে দূরবর্তী data center থেকে কম্পিউটারের hardware, software, server, storage এবং network service ব্যবহার করা যায়। National Institute of Standards and Technology (NIST) এর মতে, cloud computing এমন একটি model যার মাধ্যমে ব্যবহারকারী প্রয়োজনমতো, দ্রুত এবং ন্যূনতম ব্যবস্থাপনায় computing resource পেতে পারে। এই ব্যবস্থায় ব্যবহারকারীকে ব্যয়বহুল server বা data center নির্মাণ করতে হয় না; বরং provider-এর অবকাঠামো ব্যবহার করে প্রয়োজন অনুযায়ী service নেওয়া যায়। এ কারণে cloud computing-এ Pay as you go billing model খুব জনপ্রিয়। NIST cloud computing-এর পাঁচটি মূল বৈশিষ্ট্য নির্ধারণ করেছে — On-demand Self-service, Broad Network Access, Resource Pooling, Rapid Elasticity এবং Measured Service। সার্ভিস মডেলের দিক থেকে এটি তিন ভাগে বিভক্ত: IaaS, PaaS এবং SaaS। আবার deployment model অনুযায়ী Public, Private, Hybrid ও Community Cloud রয়েছে। Virtualization হলো cloud computing-এর মূল ভিত্তি। বর্তমানে AWS, Microsoft Azure এবং Google Cloud Platform বিশ্বের শীর্ষ cloud service provider।
English Paragraph
Cloud computing is one of the most important concepts in modern information technology. It means that computer resources such as servers, storage, software, databases, and networks are provided over the internet from remote data centers. According to NIST, cloud computing is a model that gives users convenient, on-demand access to shared computing resources with very little management effort. NIST describes five essential characteristics: On-demand Self-service, Broad Network Access, Resource Pooling, Rapid Elasticity, and Measured Service. The "Pay as you go" model is directly linked to the Measured Service characteristic. Cloud computing is divided into three service models: Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS). Based on deployment, it can be Public, Private, Hybrid, or Community Cloud. Virtualization is the foundation of cloud computing. Today, Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform are the leading providers.

BCS প্রিলিমিনারি — Verified প্রশ্ন

৪৭তম BCS৪৭তম BCS প্রিলিমিনারি · ১৯ সেপ্টেম্বর ২০২৫
ক্লাউড কম্পিউটিং এর কোন মডেলটি অ্যাপ্লিকেশন তৈরি করার জন্য প্রোগ্রামারদেরকে প্ল্যাটফর্ম সরবরাহ করে?
(ক) IaaS
(খ) SaaS
(গ) PaaS
(ঘ) DaaS
📌 PaaS (Platform as a Service) — developer-দের জন্য ready-made platform সরবরাহ করে। Infrastructure provider manage করে, developer শুধু code লেখে। উদাহরণ: Google App Engine, Heroku। একই প্রশ্ন ৪৩তম BCS-এও এসেছিল।
৪৪তম BCS৪৪তম BCS প্রিলিমিনারি · ২৭ মে ২০২২
অ্যামাজন-এর ক্লাউড প্ল্যাটফর্ম কোনটি?
(ক) Azure
(খ) AWS
(গ) Cloudera
(ঘ) উপরের সবগুলো
📌 AWS = Amazon Web Services। Amazon-এর cloud platform। বিশ্বের সবচেয়ে বড় commercial cloud provider। ২০০৬ সালে EC2 ও S3 দিয়ে যাত্রা শুরু।
৪৪তম BCS৪৪তম BCS প্রিলিমিনারি · ২৭ মে ২০২২
নিচের কোন মডেলটি Cloud Computing সেবা প্রদানকারীগণ ব্যবহার করে না?
(ক) IaaS
(খ) CaaS
(গ) PaaS
(ঘ) SaaS
📌 Cloud computing-এর তিনটি standard service model হলো IaaS, PaaS এবং SaaS। CaaS (Computing as a Service) একটি non-standard term — এটি Cloud service model হিসেবে ব্যবহৃত হয় না।
৪৩তম BCS৪৩তম BCS প্রিলিমিনারি · ২৯ অক্টোবর ২০২১
নিচের কোন প্রযুক্তি 'Pay as You Go' সার্ভিস মডেল অনুসরণ করে?
(ক) Internet of Things (IoT)
(খ) Cloud Computing
(গ) Client-Server Systems
(ঘ) Big Data Analytics
📌 Cloud Computing 'Pay as you go' মডেল অনুসরণ করে — NIST-এর Measured Service বৈশিষ্ট্যের অংশ। যতটুকু ব্যবহার করা হবে ততটুকু বিল হবে।
৪৩তম BCS৪৩তম BCS প্রিলিমিনারি · ২৯ অক্টোবর ২০২১
ক্লাউড কম্পিউটিং-এর কোন মডেলটি অ্যাপ্লিকেশন তৈরি করার জন্য প্রোগ্রামারদেরকে প্ল্যাটফর্ম সরবরাহ করে?
(ক) SaaS
(খ) IaaS
(গ) PaaS
(ঘ) HaaS
📌 PaaS = Platform as a Service। Developer-রা infrastructure manage না করে শুধু application develop করেন। এই প্রশ্নটি ৪৩তম ও ৪৭তম BCS উভয়েই এসেছে।
৪০তম BCS৪০তম BCS প্রিলিমিনারি · ২০১৯
ক্লাউড কম্পিউটার এর সার্ভিস মডেল কোনটি?
(ক) IaaS, SaaS, SaaN
(খ) IaaS, PaaS, SaaS
(গ) IaaS, PaaS, MaaS
(ঘ) PaaS, SaaS, NaaS
📌 Cloud computing-এর তিনটি service model: IaaS (Infrastructure as a Service), PaaS (Platform as a Service), SaaS (Software as a Service) — NIST-এর সংজ্ঞা অনুযায়ী।
৩৮তম BCS৩৮তম BCS প্রিলিমিনারি · ডিসেম্বর ২০১৭
ক্লাউড সার্ভার নিচের কোনটিতে সবচেয়ে ভালো বর্ণনা করা সম্ভব?
(ক) নেটওয়ার্কের মাধ্যমে যুক্ত একাধিক কম্পিউটার সার্ভার
(খ) একটি বিশাল ক্ষমতাসম্পন্ন কম্পিউটার সার্ভার
(গ) ব্যবহারকারীর চাহিদা অনুযায়ী কম্পিউটিং সেবা দেওয়া
(ঘ) কেন্দ্রীয় একটি সার্ভার যা সব ডেটা ধারণ করে
📌 Cloud server হলো এমন একটি ব্যবস্থা যা ব্যবহারকারীর চাহিদা অনুযায়ী on-demand কম্পিউটিং সেবা দেয় — এটাই NIST-এর On-Demand Self-Service বৈশিষ্ট্যের মূল ভাব।
৩৬তম BCS৩৬তম BCS প্রিলিমিনারি · জানুয়ারি ২০১৬
Cloud Computing-এ ব্যবহারকারী কোন billing পদ্ধতিতে বিল পরিশোধ করে?
(ক) মাসিক নির্দিষ্ট বিল
(খ) বার্ষিক চুক্তিতে একবারে
(গ) ব্যবহার অনুযায়ী (Pay as you go)
(ঘ) ফ্রি, কোনো বিল নেই
📌 "Pay as you go" — যতটুকু ব্যবহার করবে ততটুকু বিল। এটি NIST-এর Measured Service বৈশিষ্ট্যের সাথে সম্পর্কিত। বিদ্যুৎ বিলের মতো।
৩৫তম BCS৩৫তম BCS প্রিলিমিনারি · মার্চ ২০১৫
Cloud Computing-এর জনক (pioneer) হিসেবে কাকে বিবেচনা করা হয়?
(ক) J.C.R. Licklider
(খ) Eric Schmidt
(গ) John McCarthy
(ঘ) Tim Berners-Lee
📌 J.C.R. Licklider (Joseph Carl Robnett Licklider) ১৯৬০-এর দশকে interconnected computer network-এর ধারণা দেন — তাই তাকে cloud computing-এর জনক বলা হয়। Eric Schmidt শুধু "Cloud Computing" শব্দটি দিয়েছেন।
